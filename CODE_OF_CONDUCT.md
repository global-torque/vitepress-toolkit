# Code of conduct

Global Torque public projects require professional, technically focused
collaboration. Participants must communicate without harassment, threats,
discrimination, doxxing, deliberate disruption, or publication of private
security reports.

Maintainers may edit or remove contributions, comments, issues, or other
participation that violates these standards. Repeated or severe violations may
result in temporary or permanent exclusion from project spaces.

This policy applies in repositories, issue trackers, code review, community
forums, and public events where someone represents a Global Torque project.
Report conduct problems privately through the organization contact shown on
its GitHub profile. Security vulnerabilities must use the affected
repository's private vulnerability-reporting form.
